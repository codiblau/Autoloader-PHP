Installation

To install this script, just drop the file autoload.php on your server.

Configuration

This script has just only 3 parameters. If you open autoload.php file you can see the next parameters that you can configure:

$ignore

Variable $ignore is an array which contains folders' name to be ignored. For instance, if you have a folder called "Documentation" and you want to ignore this folder, just add in this array (See Example section). By default, just 'cgi-bin' folder is ignored.

Note: Just write the NAME of folders, not the entire path.

$postfix

This variable contains the extension of your class files. Usually, classes in php has the extension '.php' or '.class.php'. You need to configure which is your extension.

$startpath

$startpath variable is optional to be configured. This variable contains which is the "root" of your application. By default, the "root" is your "document root", but you can start the search of your classes where you want.

How to use

First of all, you need install and configure the file autoload.php. See Installation section and Configuration section.

Then, you need to configure your classes. Your files name's need to be EQUAL as class name, this means that if you have a class called "Class1" you must to rename its file to "Class1.php".

Finally, just include autoload.php like require_once('PATH_TO_AUTOLOAD/autoload.php'). If you have a "template" and you put the "require_once" in it, you can call all your classes only instantiating them! You can see an example in the Example section.

Configuration

Now, we need to configure the parameters in autoload.php.

  • $ignore: we will add the folder "Documentation" in this array. This means that autoload.php won't search in "Documentation" folder and their subfolders.
  • $postfix: our classes have the extension ".php", so this parameter must to be ".php".
  • $startpath: we want to search "Class1.php" and "Class2.php" from "root", so we can keep this parameter with 'NULL' value.
